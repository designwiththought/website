# Inline-CSS audit and BEM refactor

_2026-04-25 — branch `claude/implement-design-system-J9l0x`_

## Why

Across the design port, `style="…"` attributes had crept onto layouts,
components, and a couple of generated content snippets. That makes the
HTML noisy, ducks the cascade, and means a visual change has to be made
in N places instead of one CSS file. This pass moves every reusable
declaration into `src/css/site.css` under BEM-style names, and
keeps a single, narrow exception for a content-driven CSS custom property.

## Result

| Surface | Before | After |
|---|---|---|
| Inline `style="…"` in `src/layouts/` and `src/components/` | 38 | 0 |
| Inline `style="…"` in `dist/` | 1 600+ (every rendered SVG icon spacer × pages) | 1 (the documented MarginaliaPin custom-property exception, repeated wherever the component is used) |
| Files using inline styles | 17 | 1 |
| New CSS rules added | — | 22 blocks (~85 lines) at the end of `src/css/site.css` |

Total page count and visual output are unchanged. 80 pages still build,
all routes still 200.

## What was inline, and where it went

### 1. Reader column width

- **Before:** every per-kind cultural reader and `project.html` had
  `class="… reader" style="max-width: 760px; margin: 0 auto;"`.
- **After:** new `.reader--wide` modifier:
  ```css
  .reader--wide { max-width: 760px; margin: 0 auto; }
  ```
- **Affected:** `project.html`, `reading.html`, `music.html`,
  `movies.html`, `podcasts.html`, `bookshelf.html`.

### 2. Inline icon spacing inside link text

- **Before:** every `<svg … style="margin-right:4px">` next to "Back"
  links, and `<svg … style="margin-left:4px">` next to "See the now
  page →" / drawer Menu/Close.
- **After:** two utilities + a structural rule that handles the most
  common case automatically:
  ```css
  .reader__back svg,
  .btn--link svg { margin-right: 4px; }
  .icon--leading  { margin-right: 4px; }
  .icon--trailing { margin-left:  4px; }
  ```
- **Affected:** `base.html` (menu/close), `home.html` (now-page arrow),
  `article.html`, `note.html`, `project.html`, `reading.html`,
  `music.html`, `movies.html`, `podcasts.html`, `bookshelf.html` (back
  arrows).

### 3. Cover placeholders (per-kind readers, until covers.jsx is ported)

- **Before:** every per-kind reader had something like
  ```html
  <div class="book-card__cover-placeholder" style="aspect-ratio: 2/3;
       background: var(--paper-2, #efe6d6); display: flex;
       align-items: center; justify-content: center; padding: 18px;
       text-align: center; font-family: var(--font-display);
       color: var(--ink-800);">
  ```
  (with shape / paint variants per kind).
- **After:** a small block + paint modifier + shape modifiers:
  ```css
  .cover-placeholder {
    display: flex; align-items: center; justify-content: center;
    padding: 18px; text-align: center;
  }
  .cover-placeholder--paper {
    background: var(--paper-200);
    font-family: var(--font-display);
    color: var(--ink-800);
  }
  .cover-placeholder--book   { aspect-ratio: 2 / 3; }
  .cover-placeholder--poster { aspect-ratio: 2 / 3; }
  .cover-placeholder--square { aspect-ratio: 1 / 1; }
  ```
- The base class only sets layout. Reading uses
  `.cover-placeholder.cover-placeholder--book.reading-card__spine.reading-card__spine--cocoa`
  so the spine class still paints; bookshelf/movies/music/podcasts use
  `…--paper …--book|--poster|--square`.
- Bonus: dropped a stray reference to `var(--paper-2, #efe6d6)` which
  used a non-existent token (the actual paper-200 was masked by the
  fallback); now correctly bound to `--paper-200`.

### 4. Cover frame margins + widths

- **Before:** `style="margin: 28px 0 36px; max-width: 320px"` (or 240,
  220, depending on kind) on the wrapping div in each per-kind reader.
- **After:**
  ```css
  .cover-frame { margin: 28px 0 36px; }
  .cover-frame--book    { max-width: 320px; }
  .cover-frame--album   { max-width: 320px; }
  .cover-frame--poster  { max-width: 240px; }
  .cover-frame--podcast { max-width: 220px; }
  ```

### 5. `t-small` colour + spacing variants

- **Before:** scattered `<p class="t-small" style="color: var(--fg-muted)…">`.
- **After:** small modifier set:
  ```css
  .t-small--muted    { color: var(--fg-muted); }
  .t-small--subtle   { color: var(--fg-subtle); }
  .t-small--centered { text-align: center; }
  ```
  plus three page-specific blocks for the longer/more opinionated
  one-offs:
  ```css
  .site-footer__tagline   { color: var(--fg-muted); max-width: 36ch; margin-top: 10px; }
  .notes-archive-line     { color: var(--fg-subtle); text-align: center; margin-top: 40px; }
  .about__source-note     { color: var(--fg-muted); margin-top: 48px; }
  .colophon__signoff      { color: var(--fg-subtle); margin-top: 56px; font-style: italic; }
  .note-reader__related-date { color: var(--fg-subtle); margin-right: 10px; }
  ```

### 6. Hero aside marginalia

- **Before:** `<span class="marginalia" style="display:block;
  margin-bottom:10px; transform:rotate(-3deg);">`.
- **After:**
  ```css
  .marginalia--block       { display: block; margin-bottom: 10px; }
  .marginalia--tilt-3-neg  { transform: rotate(-3deg); }
  ```

### 7. MarginaliaPin (documented exception)

The `<MarginaliaPin>` MDX component takes a `rotate=` prop that's
content-driven (different essays will want different angles). To keep
the inline footprint as small as possible without inventing one
modifier per integer rotation, the inline `style` now only sets a
CSS custom property:

```html
<span class="marginalia marginalia--rotated" style="--marginalia-rotate: {{rotate}}deg">
```

```css
.marginalia--rotated { transform: rotate(var(--marginalia-rotate, -2deg)); }
```

This is the only `style="…"` left in the source tree. It's intentional —
one variable, one declaration, fully isolated.

### 8. Theme toggle visibility

- **Before:** `theme.js` directly mutated `style.display` on the sun /
  moon `<svg>`s, and the sun SVG carried `style="display:none"` in
  `base.html` to hide it on first paint.
- **After:** CSS handles both, driven by the `data-theme` attribute on
  `<html>` (which `theme.js` already toggles):
  ```css
  .theme-toggle__sun { display: none; }
  [data-theme="dark"] .theme-toggle__moon { display: none; }
  [data-theme="dark"] .theme-toggle__sun  { display: inline-block; }
  ```
  `theme.js` still flips the attribute; the visibility toggle is
  declarative now.

### 9. Movie reader — title-year parenthetical

- **Before:** `<em style="font-family: var(--font-serif); font-style:
  italic; color: var(--fg-muted); font-size: 0.7em;">({{year}})</em>`.
- **After:** `<em class="movie-reader__year">({{year}})</em>` plus
  matching rule.

### 10. Home — "All projects" inline-block link

- **Before:** `<a class="btn btn--link" style="margin-top: 12px; display:
  inline-block;">`.
- **After:** new `.btn--link-block` modifier sets both.

### 11. Enjoying pagination nav

- **Before:** wrapper carried `style="display: flex; justify-content:
  space-between; align-items: center; max-width: 760px; margin: 32px
  auto 0;"`.
- **After:** `.enjoying-pagination` block + `.enjoying-pagination__status`
  for the muted "Page n of N" text.

### 12. Dropcap

- **Before:** `<p class="dropcap" style="max-width:none">`.
- **After:** `.dropcap { max-width: none; }` so the override is part of
  the class.

## Naming summary (for future reference)

The block names follow the design's existing BEM cadence
(`block`, `block__element`, `block--modifier`):

| Block | Elements / Modifiers |
|---|---|
| `cover-placeholder` | `--paper`, `--book`, `--poster`, `--square` |
| `cover-frame` | `--book`, `--album`, `--poster`, `--podcast` |
| `reader` | `--wide` |
| `marginalia` | `--block`, `--tilt-3-neg`, `--rotated` |
| `t-small` | `--muted`, `--subtle`, `--centered` |
| `btn` | `--link-block` |
| `icon` | `--leading`, `--trailing` |
| `enjoying-pagination` | `__status` |
| `note-reader` | `__related-date` (new element on existing block) |
| `movie-reader` | `__year` (new element on existing block) |
| `notes-archive-line` | (one-off page block) |
| `about` | `__source-note` |
| `colophon` | `__signoff` |
| `site-footer` | `__tagline` |

Where a class only needs to exist once on the site (sign-off lines,
single source notes), it's named after its block context rather than
promoted to a generic utility.
