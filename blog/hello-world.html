Hello, World

March 20, 2026 · 6 min read

Why I decided to build a personal website in 2026, what I hope to write about, and why plain HTML still makes sense.

Every developer has written "Hello, World" a hundred times. It's the first thing you type when learning a new language, the proof that something works. This post is my Hello, World — not for a programming language, but for this little corner of the internet I'm building for myself.1

Why now?

I've been meaning to start a personal website for years. The usual story: I'd pick a framework, spend a weekend configuring it, get distracted by the tooling, and never write a single word. The stack became the project, and the writing never happened.

This time I took a different approach. No build step. No framework. No npm install. Just HTML files, a CSS stylesheet, and words. It turns out that when you remove all the friction, you actually get to the part that matters: writing.2

What I'll write about

A mix of things, honestly. I work in technology and think about software a lot, so there will be plenty of that. But I also hike, read, and have opinions about things that have nothing to do with code. This site is meant to hold all of it.

I'm organizing posts into a few types:

  • Essays — longer, more considered pieces (like this one)
  • Notes — quick thoughts, interesting links, observations
  • Logs — activity reports from hikes, reading, projects
  • Photos — visual posts, mostly from time outdoors

Why plain HTML?

There's something satisfying about a website that's just files. No server-side rendering, no hydration, no JavaScript bundle. Open the HTML in a browser and it works. Put the files on any web server and they're live. It's the simplest possible version of a website, and that simplicity is a feature.3

The trade-off is real: adding a new post means creating an HTML file and updating a listing by hand. But for the volume I expect to write — maybe a few posts a month — that's not just manageable, it's pleasant. Every post is a file I can read and edit in any text editor.

What's next

I'm going to keep this simple and just write. No posting schedule, no content strategy, no analytics to obsess over. Just thoughts, published when they're ready.

If you've been thinking about starting your own site, I'd encourage you to do it. Skip the framework decision, open a text editor, and write an index.html. You can always add complexity later. But you might find, like I did, that you don't need to.
