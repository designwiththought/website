On the joy of reading source code

March 19, 2026 · 2 min read

Sometimes the best documentation is the code itself.

I was digging through the Redis source code this morning — not looking for anything specific, just curious — and it struck me how readable it is. Clear variable names, focused functions, comments that explain why not what.1

We tell junior developers to read more code, but I don't think we do it enough ourselves. There's a particular kind of satisfaction in tracing how a well-built system handles an edge case, or seeing how someone solved a problem you've been thinking about.

It's like reading good prose. You learn the craft by studying the work of people who are better at it than you.

Anyway — if you haven't read the source of a tool you use every day, try it. You might be surprised how much you learn in 30 minutes.
