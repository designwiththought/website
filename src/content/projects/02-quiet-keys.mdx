Most keyboard-shortcut designs are inherited, undocumented, and slightly wrong. Quiet Keys is a library I started maintaining as a centralized reference for the patterns that actually work — and the ones that quietly don't.

## What's in it

Forty-odd interaction patterns, each with:

- The keybinding(s)
- When to use it (and when not)
- The screen-reader behaviour it produces
- Its escape hatch — the way out if you're caught in it
- Notes from real production use

## The goal

Stop designing keyboard shortcuts alone. Borrow from the library. Contribute back.
