## Premise

The tab order of a web page is invisible until you break it. Most bugs in keyboard accessibility come from the tab order diverging — quietly, in a code review no one noticed — from the reading order of the page.

tab-order.dev is a browser extension that draws the tab order directly onto the page as a numbered path. You click the toolbar icon, and the invisible becomes obvious.

## What it does

- Numbers every focusable element in the order the browser would visit them.
- Draws arrows between consecutive elements so the path is legible at a glance.
- Flags the common anti-patterns: `tabindex` greater than zero, focusable containers with non-focusable children, hidden elements still in the tab path.
- Exports a PNG of the annotated page, for tickets.

## Where it runs

Chrome, Firefox, and Edge — the Safari port is on a shelf waiting for a free weekend. The source is open and the extension is free. There is no analytics, no telemetry; the only reason I know how many people use it is that Chrome tells me the install count.

## What I'd do differently

I'd have shipped the "export PNG" button sooner. The thing people actually want is a quick way to put the evidence in a Linear ticket, not a debugging session. Most of the value is in the shareable artifact.
